Slam (Simultaneous Localization andMapping

Introduction Paraphrased Version

SLAM, which stands for Simultaneous Localization and Mapping, is a fundamental technique used in robotics and autonomous machines. It enables a robot or device to create a map of an unfamiliar area while at the same time figuring out where it is within that environment. This capability is crucial for systems that need to navigate without GPS or any pre-existing information about their surroundings.

To achieve this, SLAM integrates data from various sensorssuch as cameras, LiDAR, sonar, and IMUsand applies specialized algorithms to estimate the robots position and the structure of the environment. Since the robot is constantly moving, SLAM must frequently update the map and correct errors to stay accurate. Modern SLAM systems rely on advanced approaches, including Kalman filtering, particle filtering, graph-based optimization, and deep learning techniques.

This technology is used extensively in self-driving vehicles, drones, service and industrial robots, AR/VR platforms, and even mobile devices. SLAM allows these systems to navigate safely, avoid obstacles, and operate independently in changing real-world conditions.

1-Factor Graphs: GTSAM models problems as factor graphs, which represent variables and constraints (factors) in a graphical structure. This allows efficient computation of the most probable configuration of variables.

2-Bayes Networks: Instead of relying on sparse matrices, GTSAM uses probabilistic graphical models to perform inference and optimization.

3-Smoothing and Mapping (SAM): It excels at problems like SLAM (Simultaneous Localization and Mapping), where a robot builds a map while localizing itself.

**Factor Graph: A bipartite graph connecting variables and factors (constraints). Each factor encodes a measurement or prior that relates one or more variables.

Nonlinear: The relationships between variables are nonlinear (e.g., rotations, projections), requiring iterative optimization.

# Add Range-Bearing measurements to two different landmarks L1 and L2
measurement_model = gtsam.noiseModel.Diagonal.Sigmas(np.array([0.1, 0.2]))
BR = gtsam.BearingRangeFactor2D
l = {k:gtsam.symbol('l',k) for k in [1,2]} # name landmark variables
slam_graph.add(BR(1, l[1], gtsam.Rot2.fromDegrees(45),np.sqrt(4.0 + 4.0), measurement_model)) # pose 1 -*- landmark 1
slam_graph.add(BR(2, l[1], gtsam.Rot2.fromDegrees(90), 2.0,measurement_model)) # pose 2 -*- landmark 1
slam_graph.add(BR(3, l[2], gtsam.Rot2.fromDegrees(90), 2.0,measurement_model)) # pose 3 -*- landmark 2

**What Are Range-Bearing Measurements? Range: The straight-line distance from the robot to a landmark.

Bearing: The angle (usually relative to the robot's heading) to the landmark.

These measurements are typically obtained from sensors like laser rangefinders, sonar, or vision systems.**
